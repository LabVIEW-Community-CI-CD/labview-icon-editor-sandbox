## Readiness Process
- Readiness is reported per run in `reports/test-env-readiness-<run>.md` generated by CI (DoD Aggregator).
- Auto-checks: Ubuntu gate runner is validated automatically in the readiness script.
- Self-hosted/License checks: Mark readiness via environment variables in the workflow or repository secrets (values: `ready`, `warning`, `blocker`):
  - `LV2021_X64_STATUS`, `LV2021_X86_STATUS`, `LV_UTF_LICENSE_STATUS`
- The readiness report flags blockers; DoD Aggregator fails if any requirement is a blocker.
- Owners update this document when requirements or runner labels change.
